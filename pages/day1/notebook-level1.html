Why keep this log

Real engineers write down what they wired and why, so when something breaks, they can retrace their steps instead of guessing. You're doing the same thing today.

1. Wiring log

Every time you connect or move a wire, add a row. Even the ones that turned out wrong.

FromToPurposeWorked? (Y/N)

2. When something doesn't work, walk this checklist

Is the LED in the right direction? (long leg = positive)

Does the GPIO number in your code match the pin you actually wired?

Is there a complete path from GPIO, through the LED and resistor, back to GND?

Did you save the file before running it?

Does the terminal show an error message? Read it out loud, it usually tells you exactly what's wrong.
