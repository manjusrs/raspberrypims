1Deleting and copying folders

rm only deletes files. Folders need their own commands.

CommandWhat it doesExample
rmdirDeletes a folder, only if it's completely emptyrmdir old_folder
rm -rDeletes a folder and everything inside it, empty or notrm -r old_folder
cp -rCopies a whole folder and everything inside itcp -r day2 day2_backup
ls -laLists everything in detail, including hidden filesls -la
rm -r is the most dangerous command you've learned so far. There's no undo, no trash can, no "are you sure." Read the folder name twice before you press enter.

2Try it

Make a throwaway folder: mkdir scratch

Try to delete it with rmdir scratch. It should work since it's empty.

Make it again, then put a file inside it: mkdir scratch && touch scratch/notes.py

Try rmdir scratch again. It should refuse, the folder isn't empty.

Use rm -r scratch instead. This time it should work.

3While loops

A for loop repeats a fixed number of times. A while loop repeats as long as a condition stays true, you don't have to know in advance how many times.

count = 0
while count < 5:
    print(count)
    count = count + 1

4Getting input from a person

input() pauses your program and waits for someone to type something, then hands that text back to you.

name = input("What's your name? ")
print("Hello, " + name)

5Put them together

Type this into warmup.py and run it. Keep typing commands until you type exit.

command = ""
while command != "exit":
    command = input("Type a command (exit to quit): ")
    print("You typed:", command)

print("Program ended.")
This loop doesn't know ahead of time how many times it will run, that's exactly what makes it a while loop and not a for loop. This exact pattern, a loop that runs "forever" until a condition changes, is what we're using this afternoon to watch a sensor.
