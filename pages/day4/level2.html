Independent Challenge · Day 4

Build a Real Photo Booth

LEVEL 2 · ON YOUR OWN
Name:  

The challenge

Build a photo booth with a GUI window showing clickable filter buttons, Grayscale and Sepia at minimum, and a physical button wired to a GPIO pin that actually takes the picture. Whichever filter button was clicked most recently is the one applied to the next photo.

Your task

Wire a new push button to GPIO 27 and GND, this is your shutter

Build a Tkinter window with at least 2 filter buttons

Store your filters in a dictionary, name paired with function, and generate the buttons with a loop, not one at a time by hand

Pressing the physical button captures a photo, applies the currently selected filter, and shows the result in the GUI

Hint 1: two different Button classes

Both tkinter and gpiozero have a class called Button. Importing both normally will cause one to silently overwrite the other. Import one with a different name.

from gpiozero import Button as PiButton
Hint 2: building buttons from a dictionary, without a closure bug

Looping over a dictionary to create buttons has a classic trap: if your button's command directly uses the loop variable, every button ends up using whatever the loop variable was last set to, not the value it had when that specific button was created. Fix it by capturing the current value as a default argument in a lambda.

filters = {"Grayscale": ImageOps.grayscale, "Sepia": sepia}

for name, func in filters.items():
    tk.Button(window, text=name,
              command=lambda f=func: set_filter(f)).pack(side="left")
Hint 3: full structure, if you're really stuck 
import tkinter as tk
from gpiozero import Button as PiButton
from picamera2 import Picamera2
from PIL import Image, ImageTk, ImageOps
from datetime import datetime

def sepia(image):
    gray = ImageOps.grayscale(image)
    return ImageOps.colorize(gray, black="#3f2f1e", white="#f4e1c1")

filters = {"Grayscale": ImageOps.grayscale, "Sepia": sepia}
current_filter = ImageOps.grayscale

camera = Picamera2()
camera.start()

window = tk.Tk()
photo_label = tk.Label(window)
photo_label.pack()

def set_filter(f):
    global current_filter
    current_filter = f

for name, func in filters.items():
    tk.Button(window, text=name,
              command=lambda f=func: set_filter(f)).pack(side="left")

def take_photo():
    filename = datetime.now().strftime("booth_%H%M%S.jpg")
    camera.capture_file(filename)
    img = Image.open(filename)
    filtered = current_filter(img)
    display_img = ImageTk.PhotoImage(filtered.resize((320, 240)))
    photo_label.config(image=display_img)
    photo_label.image = display_img

shutter = PiButton(27)
shutter.when_pressed = lambda: window.after(0, take_photo)

window.mainloop()
Notice window.after(0, take_photo) instead of calling take_photo() directly from the button. gpiozero's button events fire on a background thread, and handing the actual work to window.after() makes sure Tkinter runs it safely on its own thread instead.
